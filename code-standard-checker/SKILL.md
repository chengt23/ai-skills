# 代码规范检查技能

此技能用于检查代码是否符合项目规范和行业最佳实践，支持多种编程语言的规范检查。

## 支持的语言

- Java
- JavaScript/TypeScript
- Python
- HTML/CSS
- 其他常见编程语言

## 检查维度

### 1. 代码风格规范
- 命名规范（类名、方法名、变量名、常量名）
- 缩进和格式化
- 注释规范（类注释、方法注释、重要代码块注释）
- 文件头注释

### 2. 最佳实践
- 代码组织结构
- 方法长度限制
- 类的单一职责
- 重复代码检测

### 3. 安全规范
- 输入验证
- SQL注入防护
- XSS防护
- 敏感信息泄露

### 4. 性能规范
- 循环中的数据库查询
- 内存泄漏风险
- 不必要的对象创建

## 检查流程

1. **读取代码文件**：获取待检查的源代码内容
2. **语法分析**：解析代码结构
3. **规范检查**：按照上述维度逐项检查
4. **问题识别**：标记不符合规范的位置
5. **生成报告**：输出详细的检查报告和改进建议
